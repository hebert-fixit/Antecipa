# Contributing to Antecipa 🤝

Thank you for your interest in contributing to **Antecipa**! We welcome contributions that improve calculations, enhance UI design, optimize performance, or expand feature sets.

---

## 🛠️ Development Workflow

1. **Fork the Repository**: Create a personal fork on GitHub.
2. **Clone Locally**: Clone your fork to your development machine.
3. **Create a Feature Branch**: Name your branch descriptively:
   ```bash
   git checkout -b feat/your-feature-name
   ```
4. **Make Changes**: Ensure all code changes adhere to clean coding standards and preserve existing calculation precision.
5. **Commit Progress**: Use [Conventional Commits](https://www.conventionalcommits.org/):
   ```bash
   git commit -m "feat: add support for selective credit card rate inputs"
   ```
6. **Push and Open a Pull Request**: Push your branch and open a PR against our `main` branch.

---

## 🎨 Design & Coding Guidelines

* **Pure Vanilla Stack**: Keep the project lightweight and zero-dependency (Vanilla ES6+ JS, raw CSS variables, semantic HTML5).
* **High-Fidelity Math**: Do not modify calculation rounding logic without verifying that installments match cent-by-cent with merchant gateway receipts.
* **Fintech Luxe UX**: Maintain the premium dark-mode, HSL color palettes, and glassmorphism styling tokens defined in `styles.css`.

Thank you for helping make Antecipa better! 💎
